#INSTALLATION:

GodSentinel installation is simple.

[Step by Step configuration]:

[1] - Upload all files in the root of your dynamic site, or in the root of your CMS(portal,forum,board etc).

[2] - Run the install.php file and choose the password to access the security system panel.

[3] - After choosing the password you will have to delete the install.php file, once you have chosen the password you will see the classic link for the automatic deletion of the installation file (click it).

[4] - Once you have completed the three previous points you can access the panel (admin.php) with the chosen password, I point out that during the installation the .htaccess file will be automatically created with the sentinel configurations, to be left unchanged.

[5] - Last and fundamental point you have to include the sentinel in the file or files that you want to protect / monitor clearly a file with extension ".php" .
                    If you are using a CMS you must include the sentinel file in a carrier file, that is a file present in all the others, for example: "config.php" , "header.php" , "footer.php" .
      
      practical example:
      Open File and write on the first line --> include("godsentinel.php");
      It is essential to include it at the top of the desired file on the first line.

#DESCRIPTION:

The security system panel is very intuitive and easy to use.
The security system is complete with filters that protect files from multiple types of remote attacks such as:
[1] SQL injection
[2] XSS (Cross site scripting)
[3] LFI ( Local File inclusion )
[4] RFI ( Remote File inclusion )
[5] Directory Listing
[6] Directory Traversal
Which can filter each request, and are individually manageable from the admin panel.
.systems: 
There is also an autonomous system for blocking IP addresses, one for managing logs,
one for protecting a single table in the database of your choice (users_table) and dedicated admin password management.
The admin panel is protected from brute force attacks on login, It also has a console inside the panel developed to identify any errors in the security system and in the server and then report them to the administrator.
.plugins: 
- NoXploiter beta: is an exploit blocker,cURL blocker and Wget blocker.
Is not aggressive.

   - CrawlerKiller beta: bot/spider/crawler blocker, it also blocks connections from Tor browser.
particular plugin.

- Shell Detector beta: able to find web shells within the same directory as GodSentinel v1.0, complete with independent diagnostics.
efficient plugin.

Of course the plugins are manageable.
.banner: 
To insert the internal banner into the page you prefer, just do a simple inclusion. example --> include("banner.php");
You have the ability to format the position with simple html tags.
